# Active Todays


현재 실행중인 스케쥴러의 일자별 현황을 알려준다. (로그인을 하면 자동으로 열리는 화면이다.)

  • 일자를 선택하고 버튼을 누르면 Tree View와 Editor View 영역에 일자별 Job 리스트가 생성된다.

TIP

PCP는 일자별로 스케쥴러 엔진(쓰레드)이 생성된다.


# (Tree) 컨텍스트 메뉴

일자별 Tree 컨텍스트 메뉴를 선택 할 수 있다.

  • 컨텍스트 메뉴 설명
    • Open Diagram : 선택된 날짜의 다이어그램 화면을 Open 한다.
    • Open Status : 선택된 날짜의 Status 화면(Dashboard)를 Open 한다.
    • Remove : 선택된 날짜의 Job 정보를 메모리에서 삭제한다. (다이어그램, Tree 에서 해당날짜가 삭제된다.)
    • Reload : 선택된 날짜의 Job 정보를 다시 Load 한다.

# (Today) Job Flow Diagram

스케줄링 중인 Job 의 현황을 다이어그램 등으로 확인 할 수 있는 화면이다. 권한 유무에 따라 Job 수행, 수정 등 Job 조작이 가능하다.

  • 화면 구성
    • 좌측 상단 Today 조회 : 특정 날짜 선택 및 조회 할 수 있다. 이미 조회된 날짜 전체를 다시 조회 할 수 있다.
    • 좌측 Tree : 조회 된 Today Job 을 날짜별로 리스트 형태로 보여 준다.
    • Filter Box : Tree View에서 입력된 문자열을 포함한 레코드를 검색한다. 숫자를 입력하면 Order-Id를 검색한다.
    • 우측 Job 다이어그램 : 날짜별 Job 현황을 다이어그램 형태로 보여준다.
    • 우측 하단 Outline : 날짜별 Job 다이어그램을 한 눈에 볼 수 있게 축소된 다이어그램을 보여 준다.

다이어그램 및 Tree 의 Job 리스트 를 통해, Job 정보를 확인하거나 여러가지 Action 을 수행 할 수 있다.

  • 확인 및 수행 가능 작업
    • Job Status 확인
    • Job Action 수행
    • Job Log 조회
    • Event Log 조회
    • 통계 조회
    • Status 조회

# Job 상태 정보

  • Job Status 란?
    • Job 스케줄링, 수행, Action 등의 결과로 설정되는 값이다. 이 값은 Job 스케줄링 조건의 일부로도 사용된다.
  • Job Status 종류
    • Not Running : Job 이 수행되지 않은 초기 상태
    • Running : Job 이 수행중인 상태
    • Complete : Job 이 정상적으로 수행종료된 상태
    • Error : Job 수행중 오류가 발생한 상태
    • Not Enough : Agent 의 가용한 Qunatity 가 Job 수행에 필요한 것 보다 작은 상태
    • No Agent : Job 이 수행 될 Agent 가 없는 상태
    • Unknown : PCP 가 Job 상태를 판단 할 수 없는 상태 (대표적으로 예로 Job 이 Running 중 인 채로 PCP 가 재기동 된 경우다.)

# Job Action

  • Job Action 이란?
    • PCP 혹은 사용자가 Job 에게 시킬 수 있는 특정 동작을 말한다.
  • Job Action 종류
    • Hold : Job 스케줄링을 멈춘다.
    • Free : Job 스케줄링을 재개 한다.
    • Skip/Set OK : 후행잡 수행등을 위해, Job 상태를 강제로 정상으로 바꾼다.
    • Run Now : 수행 되지 않은 Job 을 즉시 수행 시킨다.
    • Rerun : 이미 수행 된 Job 을 재 수행 시킨다.
    • Kill : Running 중인 Job 을 종료 시킨다. (실제 수행중인 프로세스에 kill signal 을 보낸다)
    • Stop Cyclic : Cyclic Job 의 반복 수행을 멈춘다.
    • Delete : Job 을 삭제 한다.
    • Undelete : 삭제된 Job 을 삭제 취소 시킨다.

# Job Log 화면

실제 수행중인 프로세스의 Log 를 보여준다.

  • Log 내용
    • 구분 라인 \<P> (시작시각) 을 통해 프로세스 Log의 시작과 끝 라인을 알 수 있다.
  • 화면 조작 : Auto Refresh 체크박스를 체크하면 Interval(Sec) 주기로 Log 를 조회한다.

# Event Log 화면

PCP의 스케줄링 진행 정보를 보여준다.

  • Log Format : YYYY-MM-DD hh:mm:ss.uuuuuuu [ODAT][Order ID] 내용
  • Log 내용 :
    • Excute Job : Job 수행
    • UpdateTodayJobResult (sts:{JobStatus}) : Job Status 변경 ({JobStatus} 로 변경)
    • ReEnq -> RemSec:{sec}(Enq:{hhmmss}) : 다음 Job 수행을 위한 Queueing 완료
      {sec} 후에 Job 수행예정. {hhmmss} 시각에 Queueing 수행완료
  • 화면 조작 : Log 화면과 같다.

# Context Menu

Job 컨텍스트 메뉴를 통해 선/후행 Job 으로 이동, Job Action 수행, 기타정보 조회 등을 할 수 있다.

  • 컨텍스트 메뉴 설명
    • Prev Jobs
      • MyJob#1_(MyResource#1)__[NotRun] : Resource가 MyResource#1, Job 이 MyJob#1 인 선행 다이어그램으로 이동
    • Post Jobs
      • MyJob#5_(MyResource#1)__[NotRun] : Resource가 MyResource#1, Job 이 MyJob#5 인 후행 다이어그램으로 이동
      • Hold-All : 해당 Job 의 모든 후행 Job 에 Hold Action 수행
      • Free-All : 해당 Job 의 모든 후행 Job 에 Free Action 수행
    • Hold , Free, Skip/Set OK, Run Now, Rerun, Kill, Stop Cyclic, Delete, UnDelete : Job Action 참고
    • Log : Job Log 화면 Open
    • Event Log : Event Log 화면 Open
    • Statics : 통계 화면 Open
    • Properties : Job 속성 조회 및 편집 화면 Open

# Dashboard - Job Status

날짜별 Resource, Job 현황을 표로 보여준다.

  • 화면설명.
    • 좌측 상단 : Job 별 상태 현황을 백분율로 표시
    • 우측 상단 : Resource 별 Job 상태 현황
    • 하단 : 우측 상단에 선택한 Resource 별 Job 상세 상태 정보
    • 리소스 상태 컬럼의 숫자를 클릭하면 하단에 해당하는 Job 리스트가 조회되고 개별 Job을 더블 클릭하면 Diagram 화면에서 해당 Job으로 포커스가 이동한다.
