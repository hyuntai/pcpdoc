# Variables


  • 사용 가능한 변수 목록을 조회 하거나, 사용자 변수를 추가할 수 있다.
  • Job 속성 중에 일부 항목은 변수로 정의 할 수 있다.(ex: pcpcond -ADD A-JOB-OK %ODAT%)
  • 변수는 Runtime(Argument, PostProc) 또는 Order Time(Schedule>From,To)에 실제 값으로 치환된다.
  • 변수 치환 기능을 사용할 수 있는 항목
    • General->Argument
    • Advanced->Post Proc
    • Advanced->Variables
  • 변수의 유형
    • 시스템 변수는 서버에 예약된 변수이며, Job Name(JOBNM), Oder Date(ODAT) 등 실행 시점에 확인 가능한 내용이다.
    • 사용자 변수는 기본 값과 입력 조건(캘린더, From Date/To Date)에 따라 대체되는 값(sub-value)을 등록 할 수 있다.
  • 화면 설명(항목)
    • Name : 변수명
    • Type : Date, Number, String
    • Attr : Type 에 따른 속성 (Date format 혹은 Number/String 길이)
    • Value : 기본 값
    • Sub Value : 대체 값
    • Calendar : Calendar 이름
    • From(MMDD) : 시작 날짜 (입력날짜 포함)
    • To(MMDD) : 종료 날짜 (입력날짜 포함)
  • 화면 설명(입력 값 - 예시)
    • DAYSAGO 변수의 값은 30 이다.
    • CSAT_TO 변수의 기본값은 '1530' 이고, 12월 3일에는 '1630' 이 적용된다.
    • IS_HOLI 변수의 기본값은 'No' 이고, Calendar 의 Holiday 에 설정된 날짜에는 'Yes' 가 적용된다.
    • IS_HOLI_1ST_HOY 변수의 기본값은 'No' 이고, 다음 두가지 조건을 만족하면 'Yes' 가 적용된다.
      1. Calendar 의 Holiday 에 설정된 날짜
      2. 1월 1일 ~ 6월 30일
  • 사용 방법
    • 변수를 Job의 속성에 적용하려면 변수명 앞뒤에 '%' 문자를 추가 한다.
    • ex) pcpcond -ADD A-JOB-OK %ODAT%
    • Advanced->Variable 에서는 변수 이름만 입력한다.
  • 치환 중 오류 처리
    • Job 실행 전 Validation Check 과정에서 오류 발생 시 해당 Job의 상태는 Unknown으로 변경된다.
    • Job 실행 후(PostProc) 오류 발행 시에는 무시된다.(blank로 대체)
    • 운영중 Unknown상태의 Job은 작업 담당자의 학인 후 Skip 혹은 Rerun 등의 적절한 조치가 필요하다.
