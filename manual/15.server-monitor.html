# 모니터링


# 서버 모니터링


배치 서버의 주요 프로세스 상태를 실시간으로 모니터링 한다.

  • 내부적으로는 실시간으로 이벤트를 수신하고, 화면 갱신 주기만 설정하도록 한다.
  • 를 누르면 상태 리스트 중에 필터링 상태 값을 선택할 수 있다.
  • Tran Time Check(sec)는 모니터링 대상 프로세스 혹은 쓰레드로 부터 Heartbeat나 Data 메세지가 전달되지 않으면 경고표시를 하기위한 임계 값이다.
  • 모니터링 화면에서 더이상 확인이 필요가 없는 레코드는 delete할 수 있다. 여기서 삭제는 화면에 보여주기 위한 flag일 뿐 서버 운용에는 영향을 주지 않는다.
  • 모니터링 대상 주요 프로세스/쓰레드
    • 서버 프로세스
    • Engine 쓰레드 : 일자별로 엔진 쓰레드의 상태를 모니터링 한다.
    • (클라이언트) Gateway
    • (Agent) Gateway
    • 원격 데이터 송수신 프로세스
    • 파일 전송 관리 프로세스
    • Agent : 접속 허용에 대한 정책을 설정할 수 있다.

# (ACL) Agent 접속관리


  • PCP 서버는 hostname을 key로 동시에 동일한 hostname을 가진 Agent의 접속을 허용하지 않는다.
  • 기본적으로 먼저 로그인에 성공한 Agent 세션이 존재하면, 이후 접속 시도는 거부(deny)된다.
  • 최초로 Agent가 서버 접속에 성공하면 N\A 상태로 등록이 된다.
  • 접속에 성공한 Agent 리스트는 모니터링 화면에 나타나며, 사용자에 의해 Access/Deny 상태로 변경이 가능하다.
  • Deny상태로 변경하는 과정에서 세션은 강제 종료된다.
  • 재 접속 시도 시 같은 hostname의 다른 세션이 존재 하지 않더라도, Deny상태인 Agent에 대해서는 거부 처리를 한다.
  • Agent의 Delete 기능은 Deny 상태에서만 지원한다.
